# Security Policy

## Supported versions

Security fixes are provided on the latest release line only.

## Reporting a vulnerability

Please avoid posting security issues publicly first.

Preferred path:

1. Use GitHub private vulnerability reporting if enabled for this repository.
2. If private reporting is unavailable, submit through the in-app `Report` flow and prefix the title/details with `SECURITY:`.

Include:

- App version
- macOS version
- Reproduction steps
- Potential impact

You will receive a response as soon as possible after triage.
